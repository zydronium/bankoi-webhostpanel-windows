Bankoi Webhost Panel (for Windows) Installation

Prerequisites :

1)   Windows 2000 / 2003 server with IIS installed (Web service with ASP support, FTP service).
2)   A Mysql database
3)   My ODBC (ODBC Driver for mysql) installed on server.
4)   FrontPage server extension 2002.
5)   For Awstats support : Perl 5.6.0 or later (For setup see below)
6)  For DNS support : DNS Provider for WMI on windows 2000. (It comes with windows 2003) (For setup see below)

Setup:

1)  Creating System User that will run Control Panel Site
a.      Create a USER With administrator Group With named "webhostpanel" with your password
2)    Setting Site in IIS:
1)      Create a folder where you want to install the webhostpanel say c:\webhostpanel
2)      Copy all the files/folders which are in webhostpanel folder of zip file to this folder c:\webhostpanel
3)      Create a site in IIS  with site home dir as c:\webhostpanel\www. You can choose any other port say 8077 so that it can work on all domains hosted with that server.
4)      Go to properties of this site, Under "Directory Security Tab", edit Anonymous access and authentication control, press Edit button. On this dialog Allow anonymous access and uncheck Integrated Windows authentication. Click on Edit button for account used for anonymous access. Change the user here to "webhostpanel" using Browse button. Provide password. Uncheck Allow IIS to control password checkbox.
5)      Create a folder where the sites will be created using this control panel. For example c:\webspace .
6)      Give full permission to the folder where sites will be created to this user "webhostpanel". Also give full permission to the user that is running IIS service. (IISWPG_computername on windows 2003)
7)      Give Full permission to system32 folder of the server to "webhostpanel" user .
3)   Database Settings
a.      Insert webhostpanel-1.20.sql in your mysql database.
b.    Edit db.asp in inc folder. Change below line according to your database and MYODBC driver on your server
strConnection = "driver={MySQL ODBC 3.51 Driver};server=localhost;uid=root;pwd=;database=webhostpanel;"
4) Important settings
a.  Edit params.asp in inc folder. Every line is well commented. You need to check those entire path related settings in this file.
5)   Web stats settings:
1. To  generate webstats (webalizer and awstats) you need to setup a scheduled task the command must be as follows (where c:\webhostpanel must be replaced with the original path of webhostpanel installation)
c:\webhostpanel\bin\stats.bat
2. Awstats 6.1 is packed with Banoi webhostpanel. If you want to use awstatas then Perl must be installed on your server and must be enabled on your websites. In windows 2003 there must be require to set "ALLOW" under webservice extentions in IIS MMC for perl ISAPI and CGI. these webstats can e accessed as http://domainname.tld/webstat
3. If you dont want to use awstats you can set isAwstats = "NO" in params.asp similar for webalizer stats isWebalizerStats="NO" if you dont want to use webalizer stats. Webalizer stats will be available as http://domainname.tld/stats if these are enabled.
4. We have configured GeoIP plugin for perl. Awstats use this plugin to map country name from IP. You can install this plugin as descibe on maxmind's site. If you dont want to install then you need to disable this support from awstats conf file.
6)   DNS Server support:
1. For DNS server support DNS server must be installed on same machine and DNS Provider must be installed. DNS provider comes with windows 2003 but it must be installed separately on windows 2000. We have packed necessary files in "dns provider for win2000" folder. You can install these oin windows 2000 as follows. Please do not install these on windows 2003

To install the dns provider on windows 2000,
1. Copy dnsschema.mof and dnsprov.dll to <winntdir>\system32\wbem.
2. Open command propmt and change directory to <winntdir>\system32\wbem
3. Run "mofcomp dnsschema.mof" to register the class information.
4. Run "regsvr32 dnsprov.dll" to register the dll.

That's it, please login with admin / admin, and start using the webhostpanel.
